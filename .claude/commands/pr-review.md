GitHub PR #$ARGUMENTS をレビューしてください。

## 実行手順

1. `gh pr diff $ARGUMENTS` でPRの差分を取得
2. 変更されたファイルの内容を確認（必要に応じてReadツールで詳細確認）
3. 以下の観点でレビューを実施:
   - コードの品質と可読性
   - API仕様への準拠
   - エラーハンドリング
   - セキュリティ考慮
   - テストカバレッジ
   - TypeScript型安全性
4. レビュー結果を `gh pr review $ARGUMENTS --comment` でPRコメントとして投稿

## レビューコメントのフォーマット

以下のフォーマットでPRコメントを投稿してください：

```
## PR#$ARGUMENTS レビュー

### 総評: [LGTM / 要修正 / 要確認]

[総評コメント]

---

### 良い点

[良い点を箇条書きまたはコードブロック付きで記載]

---

### 指摘事項（あれば）

[指摘事項を重要度別に記載]

---

### 確認事項

| 項目 | 状態 |
|------|------|
| API仕様準拠 | OK / NG |
| テストカバレッジ | OK / NG |
| エラーハンドリング | OK / NG |

---

### テスト結果

[テスト結果のサマリ]

[マージ可否の判断]
```

## 注意事項

- レビューは日本語で記述
- 重要な指摘がある場合は「要修正」、軽微な指摘のみの場合は「LGTM」
- コード例を含める場合は適切にフォーマット
